# RLS Fix for Public Tables Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the PostgREST/anon-key exposure on 9 `public` schema tables by enabling Row Level Security with zero policies, via an Alembic migration.

**Architecture:** A single Alembic migration (`db/migrations/versions/dda5b8820ce4_enable_rls_public_tables.py`) runs `ALTER TABLE ... ENABLE ROW LEVEL SECURITY` on all 9 flagged tables via raw `op.execute()`. A new verification script (`scripts/verify_rls_fix.py`) hits the live PostgREST endpoint with the anon key to prove the exposure exists before the migration and is closed after — this is the only verification mechanism, since the existing pytest suite runs against ephemeral SQLite and has no concept of RLS.

**Tech Stack:** Alembic (existing), `requests` (already installed, used elsewhere in `hitl/client.py` — no new dependency).

## Global Constraints

- Per the approved spec (`docs/superpowers/specs/2026-06-25-rls-fix-design.md`): zero RLS policies are added. RLS-enabled + no policies = deny-all for every role except the table owner, which is the correct end state since there is no current legitimate use case for `anon`/`authenticated` access to these tables.
- No changes to `db/session.py`, `auth/client.py`, or any other application code — this is a pure database-schema change.
- The migration's `down_revision` must be `'9cb5227a017d'` — confirmed as both the latest migration file in the repo AND the live DB's current `alembic_version` (verified via direct SQL query during brainstorming: no drift).
- Exact table list (9 tables, no more, no fewer): `benchmark_packs`, `benchmark_items`, `assessments`, `runs`, `model_responses`, `metric_results`, `scorecards`, `response_reviews`, `alembic_version`.
- This migration runs against the **real production database** (`DATABASE_URL` in `.env`, the live `afroeval` Supabase project). The actual `alembic upgrade head` step (Step 4 below) must be run and confirmed by a human or controller directly — do not let an unattended subagent apply a schema change to production without that step being visible and checkpointed.

- [ ] **Step 1: Write the verification script**

Create `scripts/verify_rls_fix.py`:

```python
"""
Verify RLS is closing the public-table PostgREST exposure.

Hits the live PostgREST endpoint directly with the anon key and reports
whether model_responses rows are readable. Run before AND after applying
the RLS migration (see db/migrations/versions/dda5b8820ce4_enable_rls_public_tables.py)
to confirm the before/after behavior change.

Usage (from afroeval/):
    .\\.venv\\Scripts\\python.exe scripts/verify_rls_fix.py
"""

import sys
from pathlib import Path

sys.path.insert(0, str(Path(__file__).parent.parent))

import requests

from api.settings import get_settings


def check_anon_access() -> None:
    settings = get_settings()
    url = f"{settings.supabase_url}/rest/v1/model_responses"
    headers = {
        "apikey": settings.supabase_anon_key,
        "Authorization": f"Bearer {settings.supabase_anon_key}",
    }
    resp = requests.get(url, headers=headers, params={"limit": 1}, timeout=15)
    resp.raise_for_status()
    rows = resp.json()
    if rows:
        print(f"EXPOSED: anon key can read {len(rows)} row(s) from model_responses — RLS is NOT blocking access.")
    else:
        print("CLOSED: anon key returned zero rows from model_responses — RLS is blocking access as expected.")


if __name__ == "__main__":
    check_anon_access()
```

```python
"""enable RLS on public tables, no anon/authenticated policies

Revision ID: dda5b8820ce4
Revises: 9cb5227a017d
Create Date: 2026-06-25 16:00:00.000000

"""
from typing import Sequence, Union

from alembic import op

revision: str = 'dda5b8820ce4'
down_revision: Union[str, None] = '9cb5227a017d'
branch_labels: Union[str, Sequence[str], None] = None
depends_on: Union[str, Sequence[str], None] = None

_TABLES = (
    "benchmark_packs",
    "benchmark_items",
    "assessments",
    "runs",
    "model_responses",
    "metric_results",
    "scorecards",
    "response_reviews",
    "alembic_version",
)


def upgrade() -> None:
    for table in _TABLES:
        op.execute(f"ALTER TABLE public.{table} ENABLE ROW LEVEL SECURITY")


def downgrade() -> None:
    for table in _TABLES:
        op.execute(f"ALTER TABLE public.{table} DISABLE ROW LEVEL SECURITY")
```
